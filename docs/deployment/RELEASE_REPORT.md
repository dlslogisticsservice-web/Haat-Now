## Rollback strategy
- **Production/main unchanged** — sprint lives on `feat/website-platform-architecture`.
- **Feature-level:** revert `d9799c4` or remove the `main.tsx` chooser — the role apps, admin, and sandbox demo
  are unaffected (the public site simply stops mounting).
- **Content-level:** `website.service.rollback(tenantId, version)` restores a prior published version instantly;
  `setStatus`/`setMaintenance` takes a site offline immediately.
- **Preview teardown:** Vercel removes the preview when the branch is deleted.
