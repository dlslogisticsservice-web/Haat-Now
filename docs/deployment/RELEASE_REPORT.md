## Rollback strategy
- **Production/main unchanged** — sprint lives on `feat/website-platform-architecture`.
- **Feature-level:** revert `a5759a9` or remove the `website` nav item + render — role apps, other admin
  consoles, and the sandbox demo are unaffected.
- **Content-level:** the Website Center **History** tab rolls back any tenant site to a prior published version
  (`website.service.rollback`); status/maintenance takes a site offline instantly.
- **Preview teardown:** Vercel removes the preview when the branch is deleted.
