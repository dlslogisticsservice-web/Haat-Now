# Driver (Captain) Onboarding Checklist — HaaT Now

For onboarding a delivery captain so they can accept jobs and get paid. COD-aware (captain
collects cash at the door). Uses existing tools (Admin → Driver workspace; Driver portal).

## 1. Account & identity
- [ ] Driver record created with name + phone (OTP login).
- [ ] Documents captured: national ID, driving licence, vehicle registration (Admin → KYC).
- [ ] Vehicle type + plate recorded.
- [ ] Background/eligibility check completed and approved.

## 2. Payout setup
- [ ] Driver wallet exists (auto-created on first credit).
- [ ] Bank/cash-out details captured for weekly driver settlement.
- [ ] Per-delivery earning / captain fee confirmed (finance).

## 3. COD readiness
- [ ] Driver briefed on **cash collection**: collect the order total in cash at delivery.
- [ ] Cash reconciliation process explained (cash owed to platform vs delivery earnings).
- [ ] Change-handling / receipt guidance provided.

## 4. App readiness
- [ ] Driver app login works; can go **online**.
- [ ] Location permission granted (realtime GPS for tracking/ETA).
- [ ] Notifications enabled (new job alerts).

## 5. Go-live verification (per driver)
- [ ] Driver appears as available; receives a test job when online.
- [ ] Accept → navigate → pick up → deliver → mark **delivered**.
- [ ] Wallet credited on delivery (`complete_delivery`); loyalty/notification events fire to customer.
- [ ] Job appears in driver history + driver settlement.

## 6. Handover
- [ ] Driver trained on the app (go online, accept, navigate, complete, earnings).
- [ ] Cash-handling policy acknowledged.
- [ ] Support + incident contact shared.
- [ ] Driver marked active.

## Reference
Admin: Driver Workspace, KYC Center, Finance (driver settlements). Portal: Driver app (jobs,
navigation, wallet/earnings).
