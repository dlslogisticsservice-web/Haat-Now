## Interpretation
- The persistence logic (mapping, validation, audit, optimistic locking, saga) adds sub-millisecond
  overhead per operation; throughput is dominated by the backend (Supabase network + Postgres) in
  production. The in-memory numbers set the ceiling for logic cost.
- Snapshot hashing is cheap enough to run inline on every save (no async crypto).
- The compensating-transaction Unit of Work scales linearly with step count, as expected.
